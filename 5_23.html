Customizing Bootstrap with Sass

Change Bootstrap's variables (colors, spacing, border-radius, breakpoints) via Sass to create a tailored design system. This page demonstrates the visual effect of overriding variables and compiling Bootstrap.

See SCSS View Components
Build locally with Sass

Example workflow: override variables, import Bootstrap, compile to CSS.

// scss/custom.scss
$primary: #ff6b6b;           // change primary color
$body-bg: #f4f7fb;          // change page background
$border-radius: .9rem;      // change default radius

// Import the Bootstrap source (installed via npm)
@import "bootstrap/scss/bootstrap";

SCSS example and build commands

Minimal example showing variable overrides and the import of Bootstrap's Sass entry point.

// scss/custom.scss
// 1) Override variables BEFORE importing Bootstrap
$primary: #ff6b6b;
$body-bg: #f4f7fb;
$border-radius: .9rem;

// 2) Optionally customize utilities / maps here
$spacers: (
    0: 0,
    1: .25rem,
    2: .5rem,
    3: 1rem,
    4: 1.5rem
);

// 3) Import Bootstrap
@import "bootstrap/scss/bootstrap";

Compile (examples):

npx sass --load-path=node_modules scss/custom.scss css/custom.css
# or (fast)
sass --load-path=node_modules scss/custom.scss:css/custom.css --style=compressed

Then include your compiled css/custom.css instead of the CDN in production.

Notes

  • Always override variables before importing Bootstrap in your main SCSS file.
  • Use a build step (sass, dart-sass, webpack, Vite) to produce the final CSS.
  • After compiling, include the compiled CSS and remove CDN if you want guaranteed consistency.
